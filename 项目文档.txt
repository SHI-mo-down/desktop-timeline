项目概述：Desktop Timeline — 桌面布局时间线管理器

一、产品比喻

像 Git 管理代码一样管理桌面图标布局。
• commit  = 自动记录布局状态
• tag     = 用户命名的检查点
• log     = 浏览布局历史
• restore = 回到任意时间点
• diff    = 查看两次布局之间的变化

二、核心逻辑

1. 时间线记录
   • 程序常驻托盘，后台轮询（2秒间隔）
   • 检测到图标位置变化 → 去抖（2秒内稳定）→ 自动记录新状态
   • 每次记录包含：时间戳、图标列表、位置坐标、布局哈希
   • 记录文件存储在 snapshots/ 目录，按时间命名

2. 时光回溯
   • 用户打开时间线面板，看到布局历史列表
   • 每条记录显示：时间、图标数量、变化摘要（"3个图标移动"）
   • 点击任意记录 → 预览布局 → 确认恢复
   • 恢复操作本身也会被记录（可撤销的撤销）

3. 检查点
   • 用户可手动创建命名检查点："工作布局"、"游戏布局"
   • 检查点高亮显示，不随自动清理删除

4. 智能合并
   • 连续的小变化自动合并为一个记录
   • 避免噪音：1像素漂移不计入变化
   • 变化阈值可配置（默认：至少3个图标移动超过5像素）

5. 跨设备导出
   • 导出单个快照为 .dslayout 文件
   • 包含：图标名称列表、坐标、网格设置
   • 导入时按名称匹配，未匹配的图标保持原位

6. 自动清理
   • 自动记录保留最近500条
   • 超过7天的自动合并为每日一条
   • 检查点永不过期

三、用户场景

   场景1：手滑点了排序
     → 程序检测到图标全变了
     → 自动记录排序前的布局
     → 托盘弹出："布局已改变，要恢复吗？"
     → 一键恢复

   场景2：尝试新布局
     → 用户保存检查点"旧布局"
     → 随意拖动图标
     → 不满意 → 回到检查点"旧布局"

   场景3：换电脑
     → 导出"工作布局.dslayout"
     → 新电脑导入
     → 图标按名称匹配恢复到近似位置

   场景4：查看历史
     → 打开时间线
     → 看到昨天下午3点自动记录的布局
     → 恢复到那个状态

四、数据格式

   时间线记录（.dslayout）：
   ┌─────────────────────────────┐
   │ # Desktop Layout Timeline   │
   │ Version: 1                  │
   │ Timestamp: 2026-06-28 15:30 │
   │ Monitor: 1920x1080, 100%    │
   │ Grid: 96x49                 │
   │ IconCount: 15               │
   │                             │
   │ 此电脑.lnk|100|200          │
   │ 回收站.lnk|200|200          │
   │ ...                         │
   └─────────────────────────────┘

五、产品定位

   不做：系统行为拦截、注入、Hook
   只做：观察 + 记录 + 恢复

   用户感知：
   "一个记住我桌面布局的小工具，误操作了可以撤销，
    换电脑了可以迁移，想怎么摆就怎么摆。"
